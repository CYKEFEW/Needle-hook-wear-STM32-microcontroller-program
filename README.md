# Needle Hook Wear Monitor Motor Control

基于 `STM32F103` 的电机控制工程，用于针钩磨损监测相关机构的驱动与张力闭环控制。

当前工程同时包含两种控制方式：

- 张力控制模式：上位机发送双通道张力值和目标转速，单片机内部执行滤波与 PID，输出力矩命令给电机驱动器。
- 速度控制模式：上位机直接发送目标转速，单片机输出步进脉冲控制电机转速。

## 工程特点

- 基于 `STM32CubeMX + HAL` 生成的 `STM32F1` 工程
- `TIM1 CH1` 输出 PWM/脉冲，驱动外部电机控制器
- `USART1` 用于接收上位机命令
- `USART2` 用于向电机/FOC 控制器发送串口命令
- 内置一阶低通、二阶陷波、误差死区和增量式 PID

## 目录结构

- `Core/Src/main.c`：系统初始化、时钟、GPIO、TIM1、USART1、USART2 初始化
- `Core/Src/usart.c`：串口命令解析与控制模式切换
- `Core/Src/control.c`：步进脉冲速度控制、方向控制、电机使能控制
- `Core/Src/pid.c`：张力闭环控制核心逻辑
- `Core/Src/filter.c`：低通、陷波、死区等滤波工具
- `Core/Src/setTorque.c`：向 FOC 控制器发送力矩或自定义命令
- `Core/Inc/main.h`：外设引脚定义、控制函数声明、PID 结构体定义
- `MDK-ARM/motorControl.uvprojx`：Keil 工程文件
- `motorControl.ioc`：CubeMX 工程文件
- `FOC_document/`：FOC 相关资料

## 硬件接口

从代码可直接确认的引脚如下：

- `PA8`：`PUL`，脉冲输出
- `PA11`：`ENA`，电机使能
- `PA12`：`DIR`，电机方向
- `PA9/PA10`：`USART1`，上位机通信
- `PA2/PA3`：`USART2`，电机/FOC 串口通信
- `PC6`：`PID_LED`，PID 状态指示

说明：

- `TIM1` 预分频为 `720-1`，用于输出控制脉冲。
- `USART1` 波特率为 `115200`，用于接收文本命令。
- `USART2` 波特率为 `115200`，用于向电机控制器发送串口命令。

## 控制模式

### 1. 张力控制模式

`ConMode 0`

进入张力控制模式后：

- 上位机先通过 `F <target>` 设置目标张力
- 再持续发送 `Data <CH1> <CH2> <target_rpm>`
- 程序在 `pid()` 内部完成：
  - 双通道张力均值计算
  - 与转速相关的陷波滤波
  - 张力低通滤波
  - 误差死区处理
  - 增量式 PID 计算
  - 输出力矩限幅
  - 通过 `USART2` 发送 `MZTxxx` 力矩命令
  - 同时下发静默转速命令

默认 PID 参数位于 `Core/Src/pid.c`：

```c
Kp = 0.015f
Ki = 0.005f
Kd = 0.001f
```

### 2. 速度控制模式

`ConMode 1`

进入速度控制模式后：

- 通过 `Con <rpm>` 直接设置电机转速
- `control()` 会根据目标转速计算 `TIM1` 的自动重装值和占空比
- 当转速低于阈值时停止 PWM 输出

## 串口命令

以下命令由 `USART1` 接收，按文本行解析，结尾使用换行符 `\n`。

| 命令 | 说明 |
| --- | --- |
| `ConMode 0` | 切换到张力控制模式，并向 FOC 发送 `MZM1` |
| `ConMode 1` | 切换到速度控制模式，并向 FOC 发送 `MX1` |
| `F <target>` | 设置目标张力，仅张力模式有效 |
| `Data <CH1> <CH2> <target_rpm>` | 输入双通道张力值和目标转速，仅张力模式有效 |
| `pid <kp> <ki> <kd>` | 在线修改 PID 参数 |
| `Con <rpm>` | 直接设置电机转速，仅速度模式有效 |
| `Forward` | 设置正转 |
| `Backward` | 设置反转 |
| `Enable` | 使能电机 |
| `Disable` | 关闭电机 |
| `focCM <cmd>` | 原样发送 FOC 串口命令到 `USART2` |

## FOC 串口输出

程序会通过 `USART2` 向外部控制器发送如下格式的命令：

- `MZT%.3f`：设置力矩
- `MZM1`：切换某种力矩/张力相关模式
- `MX1`：切换某种速度相关模式
- `focCM` 命令透传的任意字符串

其中 `MZM1`、`MX1` 的具体语义取决于你所连接的 FOC 控制器协议。

## 关键控制逻辑

`pid(float CH1, float CH2, float target_rpm)` 的主流程如下：

1. 对 `CH1`、`CH2` 取平均得到当前张力输入
2. 根据目标转速决定是否启用陷波滤波
3. 对张力信号做低通滤波
4. 将目标张力与实际张力作差得到误差
5. 对误差应用死区，减少小范围抖动
6. 使用增量式 PID 更新输出力矩
7. 对输出力矩做限幅
8. 发送力矩命令，并同步更新电机目标转速

## 当前代码中的限制

- `pid()` 内部采样频率固定写为 `50 Hz`
- 力矩输出限幅当前为 `0.0 ~ 0.05 Nm`
- `silentcontrol()` 中电机转速上限受 `rpmMAX` 限制
- 控制命令协议当前为纯文本协议，未做 CRC 或帧头校验
- `USART2` 接收逻辑目前基本未使用，主要承担发送功能

## 维护建议

- 若后续更换张力传感器，请优先重新标定 `Force_Pid.target` 的单位和死区范围
- 若转速变化引入明显机械周期扰动，可继续调整 `pid.c` 中陷波中心频率和带宽参数
- 若要提升串口稳定性，可考虑把 `USART1` 命令解析改为环形缓冲区
- 若要减少文档与代码脱节，建议后续把上位机命令协议单独整理到 `docs/` 目录
